# [Item04] 인스턴스화를 막으려거든 `private` 생성자를 사용하라

정적 멤버(field, method)로 구성된 유틸리티 클래스는 인스턴스화 불필요하다.

## Why: 왜 인스텉스를 막아야 하는가?

### 1. 상태를 가지지 않음(stateless)

객체마다 다른 상태를 갖지 않기 때문에 객체 생성 불필요하다.

### 2. 메모리 낭비

객체 생성 시 힙 메모리 할당, 가비지 컬렉션(GC) 관리 등 시스템 자원 소모되기 때문에 불필요한 자원 소모를 막아야 한다.

## How: 어떻게 차단하는가?

컴파일러가 기본 생성자를 자동으로 만들 수 없도록 `private` 생성자를 사용한다.

```java
public class Utility {

    private Utility() {}
}
```

이를 통해 클라이언트가 유틸리티 클래스 사용 시 객체 생성에 대한 오해가 없도록 **API 설계 의도를 명확히** 한다.

## When: 유틸리티 클래스는 언제 쓰는가?

**① 순수 함수 집합**  
동일한 입력값을 넣으면 동일한 결과를 반환하는 함수들을 모을 때 사용합니다.

**② 범용 객체의 보조 기능**  
특정 타입의 데이터를 다루는 공통 로직이 필요할 때 유용합니다.

- `Collection`: `Collections.sort()`, `Arrays.asList()`
- `Math`: `Math.max`, `Math.abs`, `Math.sqrt`

**③ `final` 클래스 관련 확장 메서드**  
 더이상 상속할 수 없는 `final` 클래스의 기능을 보조하거나 조작하는 메서드를 제공할 때 사용한다.

- `Stirng`: `StringUtils.isBlank()`
